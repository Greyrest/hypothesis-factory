= Руководство пользователя
:toc: left
:sectnums:
:icons: font

Дашборд: http://localhost:8088. Все действия эксперта мгновенно
пересчитывают приоритеты и сохраняются на сервере.

== Постановка R&D-задачи

В шапке дашборда:

. *Целевой KPI* — свободным текстом («снизить потери Ni и Cu в хвостах на 10%»).
. *Ограничения* — по одному в строке («не менять основную схему фабрики»).
. Галка *LLM* — использовать ли нейронку для полировки и новых гипотез.
. «▶ Прогнать все отчёты из каталога» — конвейер по всем фабрикам из данных,
  или выберите свой `.xlsx` и нажмите «Загрузить и обработать».

KPI и ограничения попадают в карточку проекта и в контекст LLM.

== Чтение результата

* *Сводка* — потери Ni/Cu и извлекаемая часть (по справке института).
* *Матрица потерь* — heatmap «поток × класс крупности», тонны Ni+Cu.
* *Находки диагностики* — сигналы с тоннами: недоизмельчение, шламовые
  потери, недофлотация, проскок классификации, пирротин. Пометка
  «(справочно)» — неизвлекаемая примесь, гипотез не порождает.
* *Карточка гипотезы* — формулировка «Если …, то …», механизм, обоснование
  с источниками (цифры отчёта + справка + литература), ожидаемый KPI
  в тоннах с явным допущением, риски, дорожная карта, оценки и приоритет.

== Что-если (контрфактуальный анализ)

Блок «🔮 Что-если»: выберите сигнал, задайте процент устранения потерь —
система посчитает адресуемый металл и эффект на KPI по ячейкам отчёта.

== Граф знаний

Граф «класс → диагноз → гипотеза → KPI» редактируется:

* выделите узлы/рёбра (клик, ctrl+клик) → «🗑 Удалить выбранное»;
* выделите ровно два узла → «🔗 Связать 2 узла» (зелёное пунктирное ребро);
* «↺ Сбросить правки» возвращает исходный граф.

Правки хранятся отдельным слоем и переживают повторную генерацию.

== Работа эксперта с гипотезами

* *Своя гипотеза* — блок «➕ Своя гипотеза»: введите текст (категорию можно
  не выбирать — определится по терминологии). Система посчитает адресуемый
  металл, подберёт обоснование и поставит карточку в общее ранжирование.
* *👍/👎* — голос по категории мероприятия; сдвигает приоритет всей
  категории (±3 за голос, потолок ±10).
* *✔ подтвердить / ✖ отклонить* — валидация: подтверждённые поднимаются,
  отклонённые уходят вниз и затемняются.
* *⚖️ Веса ранжирования* — измените веса эффекта/реализуемости/риска/новизны
  и нажмите «Применить» — вся выдача пересчитается.
