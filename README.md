# Фабрика гипотез — MVP

Прототип интеллектуального инструмента автоматической **генерации и приоритизации
технологических гипотез** снижения потерь цветных металлов с хвостами флотации.
Хакатон Норникеля, «Задача 1. Фабрика гипотез».

Вход — отчёт института о хвостах (`Хвосты *.xlsx`). Выход — ранжированные карточки
гипотез с обоснованием и цитатами, оценкой эффекта на KPI, рисками и дорожной
картой проверки + интерактивный дашборд с графом знаний.

## Запуск

```bash
bash solution/run.sh            # всё: venv -> конвейер -> открыть дашборд
bash solution/run.sh --no-llm   # принудительно без LLM
```

Требуется Python 3.10+. Зависимости: `openpyxl`, `python-docx`, `anthropic`
(ставятся автоматически в `.venv`).

**LLM-усиление**: при заданном `ANTHROPIC_API_KEY` формулировки дорабатываются и
дополняются новыми гипотезами через Claude Opus 4.8 (структурированный JSON-вывод).
Без ключа/сети система полностью работоспособна в rule-based режиме — важное
свойство для локального развёртывания (требование безопасности из ТЗ).

Результаты: `solution/output/web/index.html` (дашборд), рядом — `hypotheses_*.json`,
`hypotheses_*.csv`, `report_*.md`, `evaluation.json`.

## Архитектура

```
xlsx отчёт ──▶ parse_tailings.py   потоки × классы крупности × минеральные формы,
                    │              извлекаемый/неизвлекаемый металл; устойчив к #REF!,
                    │              пропускам, 1-2 потокам, сводным блокам
                    ▼
              diagnose.py          правила физики обогащения -> находки с тоннами:
                    │              недоизмельчение (закрытый в крупном), шламы
                    │              (раскрытый в -10), недофлотация, проскок
                    │              классификации, пирротиновый поток
                    ▼
knowledge_base.py ──┤              база знаний: справка института (чанки с
  справка (docx)    │              цитированием), каталог 26 практик из мозговых
  4 эталона (docx)  │              штурмов 4 фабрик (с метаданными), 7 правил
  правила + лит-ра  │              с привязкой к литературе; лексический retrieval
                    ▼
              generate.py          rule-based: сигналы × каталог -> карточки
                    │              (адресуемый металл считается по ячейкам отчёта,
                    │              без задвоений) + гипотезы для «пробелов»;
                    │              опционально Claude Opus 4.8 полирует и расширяет.
                    │              Ранжирование: 0.40·эффект + 0.30·реализуемость +
                    │              0.20·(1-риск) + 0.10·новизна + обратная связь
                    │              эксперта + штраф за однотипность
                    ▼
   export_results.py + build_web.py
        JSON / CSV / Markdown      дашборд: сводка, матрица потерь (heatmap),
                                   граф знаний (класс -> диагноз -> гипотеза,
                                   vis-network), карточки, 👍/👎 -> feedback.json
```

Обратная связь замыкает цикл: `👍/👎` на дашборде -> «⬇ feedback.json» ->
`python solution/pipeline.py --feedback feedback.json` — оценки эксперта
сдвигают приоритеты категорий при переранжировании.

## Соответствие ТЗ

| Требование ТЗ | Реализация |
|---|---|
| Приём структурированных данных | парсер xlsx с обработкой неполных/битых данных (#REF! у КГМК восстанавливается из форм) |
| Парсинг текстовой литературы | справка и эталонные docx -> чанки/каталог; PDF-учебники — метазаписи источников |
| Извлечение сущностей и связей | потоки, классы, минеральные формы, извлекаемость; граф знаний на дашборде |
| Паттерны и пробелы | сигналы диагностики; «пробельные» гипотезы для сигналов без практик в каталоге |
| Аналогии | перенос практик между фабриками (каталог собран со всех 4 примеров) |
| Ранжирование (новизна, реализуемость, эффект, риски) | взвешенный приоритет, все компоненты видны в карточке |
| Текстовое обоснование + цитирование | evidence: цифры отчёта + справка + правила с источниками (Абрамов и др.) |
| Механизм влияния, KPI, риски, дорожная карта | в каждой карточке; KPI — диапазон тонн с явным допущением |
| Визуализация (графы связей) | vis-network: класс крупности -> диагноз -> гипотеза; heatmap потерь |
| Экспорт PDF/DOCX/CSV/JSON | MD (печать в PDF из браузера), CSV, JSON |
| Обратная связь эксперта | 👍/👎 -> feedback.json -> переранжирование |
| Интерпретируемость | каждое число прослеживается до ячейки отчёта; допущения выписаны |
| Производительность | полный прогон 4 фабрик < 10 с (rule-based), ~1 мин с LLM |
| Локальное развёртывание | оффлайн rule-based режим, никаких обязательных внешних сервисов |
| Мультиязычность | данные/вывод RU; EN/CN — через LLM-слой (промпт), в бэклоге |

## Проверка качества

Leave-one-out по темам против эталонных мозговых штурмов экспертов
(`solution/output/evaluation.json`): покрытие тем эталона — **КГМК 67%,
НОФ Вкр 100%, НОФ мед 83%, ТОФ 100%**. Непокрытые темы — специфичные
реагентные/аппаратные решения, которых нет в данных отчёта (например, Finfix
без сигнала шламов).

Физический контроль: адресуемый металл гипотезы = объединение ячеек отчёта
(поток × класс × форма), поэтому никогда не превышает фактические потери.

## Ограничения и развитие

- Оценка эффекта — допущение по категориям (диапазон), требует калибровки на
  исторических парах «мероприятие -> эффект».
- PDF-учебники индексируются как источники, но не как полнотекстовый RAG
  (следующий шаг: векторный индекс страниц).
- Привязка гипотез к узлам конкретной схемы цепи аппаратов (png-схемы фабрик) —
  в бэклоге: распознавание схем -> граф оборудования.
